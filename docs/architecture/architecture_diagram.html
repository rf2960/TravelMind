📊 Key Flows

1️⃣ New Plan Flow (Standard)

User Input → API → Orchestrator → Parse Constraints → Recommend Destinations

  • High-Level: Return 5 recommendations → User selects → Generate itinerary
  • Medium: Return top destination + basic itinerary (no enrichment)
  • Full: Return top destination + itinerary + detailed enrichment

2️⃣ Plan Enhancement Flow

User provides existing plan → API → Orchestrator → Parse Constraints → Plan Enhancer → Add details, suggestions, costs → Return enhanced plan

3️⃣ Refinement Flow

User adjusts constraints (budget, pace, interests) → API → Orchestrator → Merge with original constraints → Regenerate plan

4️⃣ Alternatives Flow

User rejects destination → Track rejection → Update constraints → Re-run Recommender → Return 5 new options

5️⃣ Progressive Enrichment Flow (Streaming)

Itinerary ready → Call Dual Model Enricher → Stream enriched activities via SSE → Real-time updates in UI

🎯 Architecture Highlights

  • Multi-Agent System: 6 specialized LLM agents with distinct responsibilities
  • Progressive Refinement: Users can iteratively improve plans with adjustments
  • Flexible Detail Levels: High-level recommendations, medium itineraries, or full detailed plans
  • Plan Enhancement Mode: Users can upload existing plans for AI-powered improvements
  • Streaming Enrichment: Real-time progressive updates for better UX
  • Robust Error Handling: Validation, conflict detection, graceful degradation
  • URL Validation: Automatic filtering of broken/invalid links
  • Debug Mode: Comprehensive execution traces for transparency
